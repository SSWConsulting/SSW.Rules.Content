Creating Power BI Template Apps is an efficient way to manage and distribute Power BI reports and dashboards with other external users.

Once the Template App is created it will live in the [Power BI Apps](https://app.powerbi.com/groups/me/getapps/apps) marketplace. Users have the ability to use the template app with their own data or use the default data provided.

<endIntro />

### ✅ Pros

* **Ease of Distribution:** Template apps provide an easy way to distribute Power BI content to external users. Once published to the Power BI service, these apps can be shared widely without needing to individually manage access permissions
* **Ease of Maintainance:** Template apps simplify the process of updating and maintaining the Power BI content with their Release Management
* **Data Source Flexibility:** Users can easily integrate their own data
* **Trial with Sample Data:** Testing the app with sample data allows users to easily explore and understand features before using their own data. As an added bonus this is also great for demos

### ❌ Cons

* **Dependency on Power BI Environment:** External users need to have access to Power BI (Pro or Premium license), which may limit the accessibility for some users. To create Template Apps a Power BI Pro license is necessary
* **Initial Setup Complexity:** Setting up a template app for external sharing require more overhead to set it up
* **Limited Customization for End Users:** While creators can customize the app, end users have limited ability to modify or interact with the content beyond basic filtering and slicing
* **Performance Considerations:** Large datasets or complex visualizations may impact the performance and loading times for end users

### Creating Power BI Template Apps

#### Prerequisite

* Power BI Pro license
* Power BI Desktop (Windows only)
  * Mac users - either borrow Windows machine, or run Windows VM on **[VMware Fusion](https://www.youtube.com/watch?v=LWXO4DhQRL0)** (free - recommended ⭐) or Parallels
* Cloud Hosted sample database with dummy data and **read** access
  * Keep it small
* Power BI report (.pbix) file to connect sample data with

In [app.powerbi.com](https://app.powerbi.com):

1. All workspaces (left side panel) | New workspace
2. Set Name & Description

#### Step 2: Prepare the Report with Sample Data

On [Power BI Desktop](https://www.microsoft.com/en-us/power-platform/products/power-bi/desktop) (Windows only):

1. Open your `.pbix` file
2. Home | Transform Data. This will open up a new window

   <imageEmbed alt="Power BI transform data image here" size="large" showBorder={false} figurePrefix="none" figure="Power BI Desktop app" src="/uploads/rules/powerbi-template-apps/powerbi-transform-data.png" />
3. New Window Opened | Home | Manage Parameters
   (Here you can add new parameters, setup their default value or edit existing parameter names and value)

   <imageEmbed alt="Power BI manage parameters window image here" size="large" showBorder={false} figurePrefix="none" figure="New window showing the queries, parameters and source settings" src="/uploads/rules/powerbi-template-apps/powerbi-manage-parameters.png" />
4. You would need your SQL Server and Database Name and default them to your sample database details

   <imageEmbed alt="Power BI manage parameters popup imager here" size="large" showBorder={false} figurePrefix="none" figure="Edit / create parameters for your report" src="/uploads/rules/powerbi-template-apps/powerbi-manage-parameters-popup.png" />
   * Back in the Transform Data window you will have new parameters. You can see them in the left pane
5. Next you need to setup the source of each query to the these parameters. Select Query from left pane | Select Applied Steps from right pane | Select Source | Update the fx input box to parameters name (not "hardcoded strings") eg. from `Sql.Database("sql-server.net", “demo-db”)` to `Sql.Database(Server, Database)` in the right pane, and then you need to make sure every query that has source in the applied steps has paramerter names not the hardcoded string values

   <imageEmbed alt="Power BI transform data parameters image here" size="large" showBorder={false} figurePrefix="none" figure="Left pane showing Queries, right pane showing Source and middle function input box for parameters" src="/uploads/rules/powerbi-template-apps/powerbi-transform-data-parameters.png" />
6. Close and Apply. It will refresh the whole report
7. Once done, You need to publish the .pbix to the template app workspace. Home Tab | Click Publish | Select your Template App workspace, (you need to sign in with the same account of where the workspace is created) | hit publish, if you arleady has a report in that template workspace, it will warn you that it's going to override, do you want to continue, press yes, then it will upload the report for you

   <imageEmbed alt="create workspace image here" size="large" showBorder={false} figurePrefix="none" figure="Create workspace wizard with Name and Description input fields" src="/uploads/rules/powerbi-template-apps/publish-report-to-workspace.png" />
8. You can refresh the template app workspace on app.powerbi.com and report will be visible

#### Step 3: Create and Configure the Template App

1. Within the workspace, create your app by clicking "Create App"
   ![](/uploads/rules/powerbi-template-apps/create-app.png)
2. This will open up a form, fill out all the fields
   ![](/uploads/rules/powerbi-template-apps/create-app-form.png)
3. In the Navigation pane, it will automatically select the report to be included in the app
   ![](/uploads/rules/powerbi-template-apps/create-app-select-reports.png)
4. In the Parameters pane, it will show the parameter you have setup in the desktop app. Click required and then you  need to fill in the same value you filled in the desktop app and also example for user input (e.g. Server: `sql-demo-database.windows.net` and Example for user: `sql-xxx.database.windows.net`)

   <imageEmbed alt="create workspace image here" size="large" showBorder={false} figurePrefix="none" figure="Create workspace wizard with Name and Description input fields" src="/uploads/rules/powerbi-template-apps/create-app-database-parameters.png" />
5. In the auth section, it will be auto selected to SQL server and select the authenctication method to basic,
6. then in Access tab, you can add acces to other users in your org for testing or select the entire org
7. Once everything is filled out, click Save changes and then Create App
8. It will take a moment and then will give you the link to test internally in your org
9. Once everything is filled out, click the "Create app" button
   ![](/uploads/rules/powerbi-template-apps/create-app-button.png)
10. **Manage the App**
    Manage your app in the "Release Management" pane. Here you can:
    * Update the app once you have new changes in the report by clicking "Create App"
    * Share the app to specific people to test it out by clicking "Get Link"
    * Publish the app by clicking "Promote app"

      ![](/uploads/rules/powerbi-template-apps/release-management.png)

1. After you tested internally, you can promote it to pre-production in release amangemnt on power bi workspace

#### User Setup for Installed Apps

When users install and open the app they can browse the app with the dummy data from the default connection.
The user will have the option to connect their data by following the following steps:

1. Select "Connect your data" in the message at the top of the page

![](/uploads/rules/powerbi-template-apps/app-connect-data.png)

1. Enter details for the "Server" and "Database" parameters in the pop-up

![](/uploads/rules/powerbi-template-apps/app-connect-data-enter-details.png)

1. Authenticate to the database

![](/uploads/rules/powerbi-template-apps/app-connect-data-authenticate.png)

1. Once authenticated, users can start exploring the app with their own data 🚀
