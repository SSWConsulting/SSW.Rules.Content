Power BI Template Apps are an efficient way to package, distribute, and maintain Power BI reports and dashboards for external users.

Once published, a Template App lives in the [Power BI Apps](https://app.powerbi.com/groups/me/getapps/apps) marketplace, where users can install it and either explore it with the default sample data or connect it to their own data.

<endIntro />

### ✅ Pros

* **Ease of distribution:** Once published to the Power BI service, a template app can be shared widely without having to manage access permissions for each user individually
* **Ease of maintenance:** Built-in Release Management makes it simple to update and maintain your Power BI content over time
* **Data source flexibility:** Users can connect their own data with minimal effort
* **Trial with sample data:** Shipping with sample data lets users explore the features before connecting their own — and it's great for demos

### ❌ Cons

* **Dependency on Power BI:** External users need access to Power BI (a Pro or Premium license), which can limit accessibility for some. Creating template apps also requires a Power BI Pro license
* **Initial setup complexity:** Configuring a template app for external sharing takes more upfront effort
* **Limited customization for end users:** End users can filter and slice the data, but they can't modify the reports beyond that
* **Performance considerations:** Large datasets or complex visuals can slow loading times for end users

### Creating Power BI Template Apps

#### Prerequisites

* Power BI Pro license
* Power BI Desktop (Windows only)
  * Mac users - either borrow a Windows machine, or run a Windows VM on **[VMware Fusion](https://www.youtube.com/watch?v=LWXO4DhQRL0)** (free - recommended ⭐) or Parallels
* A cloud-hosted sample database with dummy data and **read** access
  * Keep it small
* A Power BI report (.pbix) file to connect to the sample data

#### Step 1: Create the Workspace

In [app.powerbi.com](https://app.powerbi.com):

1. All workspaces (left side panel) | New workspace
2. Set a Name and Description

   <imageEmbed alt="create workspace image here" size="large" showBorder={false} figurePrefix="none" figure="Create workspace wizard with Name and Description input fields" src="/uploads/rules/powerbi-template-apps/create-workspace.png" />
3. Advanced section | Workspace type | **Power BI Pro**

   <imageEmbed alt="workspace type " size="large" showBorder={false} figurePrefix="none" figure="Select Workspace type options" src="/uploads/rules/powerbi-template-apps/workspace-type.png" />
4. Advanced section | Template apps | tick **“Develop template apps”**

   <imageEmbed alt="develope template apps image" size="large" showBorder={false} figurePrefix="none" figure="Develop template apps option must be checked" src="/uploads/rules/powerbi-template-apps/develop-template-apps.png" />
5. Click Apply

#### Step 2: Prepare the Report with Sample Data

In [Power BI Desktop](https://www.microsoft.com/en-us/power-platform/products/power-bi/desktop) (Windows only):

1. Open your `.pbix` file
2. Home | Transform Data - this opens a new window

   <imageEmbed alt="Power BI transform data image here" size="large" showBorder={false} figurePrefix="none" figure="Power BI Desktop app" src="/uploads/rules/powerbi-template-apps/powerbi-transform-data.png" />
3. In the new window: Home | Manage Parameters
   * Here you can add new parameters, set their default values, or edit existing parameter names and values
   <imageEmbed alt="Power BI manage parameters window image here" size="large" showBorder={false} figurePrefix="none" figure="New window showing the queries, parameters and source settings" src="/uploads/rules/powerbi-template-apps/powerbi-manage-parameters.png" />
4. Create parameters for your **SQL Server** and **Database Name**, and default them to your sample database's details

   <imageEmbed alt="Power BI manage parameters popup imager here" size="large" showBorder={false} figurePrefix="none" figure="Edit / create parameters for your report" src="/uploads/rules/powerbi-template-apps/powerbi-manage-parameters-popup.png" />
   * Back in the Transform Data window, the new parameters appear in the left pane
5. Point each query's source at these parameters:
   * Select a query in the left pane
   * Under **Applied Steps** (right pane), select **Source**
   * In the formula (fx) bar, replace the hardcoded values with your parameter names - e.g. change `Sql.Database("sql-server.net", “demo-db”)` to `Sql.Database(Server, Database)`
   * Repeat for **every** query whose Source step still uses hardcoded strings
   <imageEmbed alt="Power BI transform data parameters image here" size="large" showBorder={false} figurePrefix="none" figure="Left pane showing Queries, right pane showing Source and middle function input box for parameters" src="/uploads/rules/powerbi-template-apps/powerbi-transform-data-parameters.png" />
6. Click **Close & Apply** - the whole report will refresh
7. Publish the `.pbix` to your template app workspace:
   * Home tab | **Publish**
   * Sign in with the same account that owns the workspace
   * Select your template app workspace and click Publish
   * If a report already exists there, confirm the overwrite when prompted — the report then uploads
   <imageEmbed alt="Publish report to workspace image here" size="large" showBorder={false} figurePrefix="none" figure="Select your template app workspace to publish your report" src="/uploads/rules/powerbi-template-apps/publish-report-to-workspace.png" />
8. Refresh the workspace in app.powerbi.com - your report will appear

#### Step 3: Create and Configure the Template App

Back in [app.powerbi.com](https://app.powerbi.com):

1. In the workspace, click **Create app**
   ![](/uploads/rules/powerbi-template-apps/create-app.png)
2. Fill out every field in the form
   ![](/uploads/rules/powerbi-template-apps/create-app-form.png)
3. On the **Navigation** pane, your report is selected automatically for inclusion in the app
   ![](/uploads/rules/powerbi-template-apps/create-app-select-reports.png)
4. On the **Parameters** pane, you'll see the parameters you set up in Power BI Desktop. For each one:
   * Tick **Required**
   * Enter the same value you used in Desktop
   * Add an example to guide users - e.g. Server value `sql-demo-database.windows.net`, example for user `sql-xxx.database.windows.net`
   <imageEmbed alt="create workspace image here" size="large" showBorder={false} figurePrefix="none" figure="Create workspace wizard with Name and Description input fields" src="/uploads/rules/powerbi-template-apps/create-app-database-parameters.png" />
5. In the **Authentication** section, the source defaults to SQL Server - set the authentication method to **Basic**
6. In the **Access** tab, grant access to other users in your org for testing, or select the entire organisation
7. Click **Save changes**, then click **Create app**
   ![](/uploads/rules/powerbi-template-apps/create-app-button.png)
8. After a moment, Power BI gives you a link to test the app internally within your org
9. **Manage the app** in the **Release Management** pane, where you can:
   * **Update** the app with new report changes by clicking **Create app**
   * **Share** the app with specific people for testing by clicking **Get link**
   * **Publish** the app by clicking **Promote app**

     ![](/uploads/rules/powerbi-template-apps/release-management.png)

#### Step 4: Publishing to Partner Center

1. After internal testing, promote the app to **pre-production** in Release Management
2. Copy the pre-production link it generates, then log in to Partner Center and either create a new offer or open an existing one
3. Fill in the required details
4. In the **Technical Configuration** section, paste the pre-production link from Release Management

   <imageEmbed alt="Partner center technical configuration image here" size="large" showBorder={false} figurePrefix="none" figure="Microsoft Partner Center | <your template app offer> | Technical configuration" src="/uploads/rules/powerbi-template-apps/partner-center-technical-configuration.png" />
5. Click **Save draft**, then **Review and publish**

<boxEmbed
  body={<>
    Once publishing starts, it takes a couple of hours to generate the preview link. After you go live to production, allow another 5–6 hours for the offer to be published and for AppSource to pick up and verify your template app.
    
    Once verified, users can install your template app from the marketplace and connect it to their own data.
  </>}

#### User Setup for Installed Apps

When users install and open the app, it opens with the default sample data. To connect their own data:

1. Select **Connect your data** in the banner at the top of the page

   <imageEmbed alt="create workspace image here" size="large" showBorder={false} figurePrefix="none" figure="Create workspace wizard with Name and Description input fields" src="/uploads/rules/powerbi-template-apps/app-connect-data.png" />
2. Enter the **Server** and **Database** parameter values in the pop-up

   <imageEmbed alt="create workspace image here" size="large" showBorder={false} figurePrefix="none" figure="Create workspace wizard with Name and Description input fields" src="/uploads/rules/powerbi-template-apps/app-connect-data-enter-details.png" />
3. Authenticate to the database

   <imageEmbed alt="create workspace image here" size="large" showBorder={false} figurePrefix="none" figure="Create workspace wizard with Name and Description input fields" src="/uploads/rules/powerbi-template-apps/app-connect-data-authenticate.png" />
4. Once authenticated, users can explore the app with their own data 🚀
