Users miss important updates when notification buttons are hidden, inconsistent, or unclear. A standard notification button pattern ensures users instantly recognize and access alerts.

<endIntro />

## Place notification buttons next to user avatars

Position the notification button beside the user avatar. Websites that use a header should put them in top-right corner and those that use a sidebar should put them in the bottom-left corner.

This follows common UI conventions and groups account-related actions logically.

## Use a bell icon for notifications

The bell is universally recognized as the icon for notifications.

Some platforms use other icons such as envelopes, inboxes, messages, or even text-only labels, but this should be avoided for regular notification buttons. 

<imageEmbed alt="Image" size="small" showBorder={true} figurePrefix="bad" figure="Non-standard notification icons." src="/uploads/rules/consistent-notifications/notification-icons-bad-examples.png" />

## Show a notification count badge

Display a small badge on the bell icon to show unread notifications.

Best practices:

* Use a red (or brand-colored) circular badge  
* Position it at the top-right of the bell  
* Show a number (e.g. ‘3’, or ‘99+’ for large counts)

<imageEmbed alt="Image" size="medium" showBorder={true} figurePrefix="ok" figure="GitHub displays a badge for notifications, but doesn't say how many there are." src="/uploads/rules/consistent-notifications/github-no-count-badge.png" />

A consistent bell icon next to the user avatar with a visible badge ensures users never miss important updates.
