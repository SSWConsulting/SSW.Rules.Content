Conversations tend to start wherever it is easiest, like a 1:1 message or a reply in another chat. The problem is that when the topic already has a group chat, discussing it anywhere else splits the conversation. Decisions get made where half the people can't see them, and newcomers can't find the history.

Before you message someone about a topic, check whether a group chat for it already exists. If it does, put a link to that chat in your message so the reader is one click away from the full context.

<endIntro />

Telling someone "there's a chat about this" without a link forces them to hunt through [Teams search](/how-to-use-teams-search), guessing at the chat name. A link takes them straight there.

Follow these steps:

1. **Check for an existing group chat** before starting a new conversation about the topic
2. **If one exists**, link to it and [add the person to the chat](/teams-group-chat) if they should be a participant
3. **If none exists**, [create a group chat](/when-to-create-a-group-chat) with everyone involved

To get the link in Microsoft Teams, hover over the chat in the chat list, click the `Meatball menu (...)` | `Copy link`. Pasting the link into Teams shows the chat title, so the reader knows where they are headed before they click.

This also keeps the number of chats down. Every duplicate chat about the same topic is one more place where context gets lost and one more conversation the team has to keep an eye on.
