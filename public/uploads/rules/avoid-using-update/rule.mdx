<introEmbed
  body={<>
The Update method on an entity in EF Core marks all of its fields as dirty. This will result in all the fields being written back to the database.
  </>}
/>
Writing the entire record to the database can cause locking issues in the database server if there are foreign key relationships involving the entity being modified.

<figureEmbed figureEmbed={ {
  preset: "badExample",
  figure: "```cs\nvar entity = context\n    .Products\n    .FirstOrDefault(item => item.ProductID == id);\n\nif (entity != null)\n{\n    entity.Name = \"New name\";\n    context.Products.Update(entity);\n\n    context.SaveChanges();\n}\n```\n\nFigure: Bad example - The whole record is written back to the database.",
  shouldDisplay: true
} } />


<figureEmbed figureEmbed={ {
  preset: "goodExample",
  figure: "```cs\nvar entity = context\n    .Products\n    .FirstOrDefault(item => item.ProductID == id);\n\nif (entity != null)\n{\n    entity.Name = \"New name\";\n    context.SaveChanges();\n}\n```\n\nFigure: Good example - Only the modified fields are written back to the database.",
  shouldDisplay: true
} } />
