We all know that jQuery is very powerful and can do a lot of stuff, including style changes. However, this practice is bad.

<endIntro />

Instead of changing the appearance of elements with jQuery, you should use CSS classes. This way you can keep all your CSS in your CSS files, specify different appearances for different classes.

It's OK to use jQuery (or vanilla JS) to add or remove classes. For this we can use `.addClass()` and `.removeClass()` methods.

```javascript
// Bad example - inline style change via jQuery
$('#element').css('color', 'red');
$('#element').css({ 'background-color': '#fff', 'font-size': '14px' });
```

```javascript
// Good example - toggle a CSS class instead
$('#element').addClass('highlighted');
$('#element').removeClass('highlighted');
```

## Modern approaches

In modern JavaScript projects, the same principle applies regardless of whether you use jQuery, vanilla JS, or a framework.

### CSS classes and CSS variables

Prefer toggling classes or updating CSS custom properties (variables) over setting inline styles:

```javascript
// Vanilla JS - toggle a class
document.getElementById('element').classList.toggle('highlighted');

// CSS variables - update a design token
document.documentElement.style.setProperty('--accent-color', '#ff5722');
```

### Component-based frameworks (React, Vue, Angular)

In React, Vue, and Angular, **avoid direct DOM style manipulation entirely**. Instead, let state drive the appearance:

```jsx
// Bad example (React) - direct DOM manipulation
document.getElementById('card').style.display = 'none';

// Good example (React) - state-driven rendering
const [isVisible, setIsVisible] = useState(true);
return isVisible ? <Card /> : null;
```

Use utility-first CSS frameworks (like [Tailwind CSS](https://tailwindcss.com/)) or CSS Modules to scope styles to components and keep styling logic out of JavaScript.

### When direct style manipulation is acceptable

There are cases where setting inline styles via JavaScript is the right tool:

- Dynamically calculated values that cannot be known at build time (e.g., element dimensions for animations, positioning a tooltip at cursor coordinates)
- CSS-in-JS libraries (styled-components, Emotion) that manage this safely

For everything else, prefer classes and let CSS do the styling.
