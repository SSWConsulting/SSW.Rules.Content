We all know that jQuery is very powerful and can do a lot of stuff, including style changes. However, mixing style logic into your JavaScript is a bad practice — it creates tight coupling between your JavaScript and visual presentation, making maintenance harder.

<endIntro />

Instead of changing the appearance of elements directly, you should use **CSS classes**. This keeps all your styles in CSS files and makes it easy to specify different appearances for different states.

It's OK to use jQuery to add or remove classes. For this, use the `.addClass()` and `.removeClass()` methods:

```javascript
// Bad example – directly manipulating inline styles with jQuery
$('#myElement').css('color', 'red');
$('#myElement').css({ 'font-size': '14px', 'background-color': '#fff' });
```

```javascript
// Good example – toggling CSS classes with jQuery
$('#myElement').addClass('is-error');
$('#myElement').removeClass('is-loading');
```

```css
/* CSS file – styles are defined once and reused */
.is-error { color: red; }
.is-loading { opacity: 0.5; }
```

## Modern Alternatives

In modern JavaScript applications, jQuery's `.addClass()` / `.removeClass()` pattern translates to equivalent native and framework-specific approaches:

### Vanilla JavaScript (no jQuery needed)

```javascript
// Native classList API – no jQuery required
document.getElementById('myElement').classList.add('is-error');
document.getElementById('myElement').classList.toggle('is-active');
```

### React

In React, manage dynamic styling by toggling class names based on component state:

```jsx
// React – conditional class names
function Button({ isLoading }) {
  return (
    <button className={isLoading ? 'btn btn--loading' : 'btn'}>
      Submit
    </button>
  );
}
```

For more complex scenarios, use a library like [clsx](https://github.com/lukeed/clsx) or [classnames](https://github.com/JedWatson/classnames).

### Tailwind CSS

With utility-first frameworks like [Tailwind CSS](https://tailwindcss.com/), toggle utilities rather than writing custom CSS:

```jsx
// React + Tailwind – toggle utility classes
<button className={`px-4 py-2 ${isLoading ? 'opacity-50 cursor-not-allowed' : 'hover:bg-blue-600'}`}>
  Submit
</button>
```

### CSS-in-JS (styled-components / Emotion)

Component frameworks commonly use CSS-in-JS for scoped styles. Avoid inline `style` props just as you would avoid `.css()` in jQuery:

```jsx
// styled-components – styles defined with the component, not inline
import styled from 'styled-components';

const Button = styled.button`
  background: ${props => props.primary ? 'blue' : 'white'};
  color: ${props => props.primary ? 'white' : 'blue'};
`;
```

## Accessibility Considerations

When toggling visual states, also update ARIA attributes to keep assistive technologies in sync:

```javascript
const el = document.getElementById('myMenu');
el.classList.toggle('is-open');
el.setAttribute('aria-expanded', el.classList.contains('is-open'));
```
