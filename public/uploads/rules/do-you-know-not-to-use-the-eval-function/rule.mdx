The Javascript command `eval` evaluates the content of a text string and then runs it as JavaScript code. It's common to see it around, however `eval` is one of the most inefficient and dangerous constructs that JavaScript has. There are always more efficient and safer ways to code and get a direct reference.

<endIntro />

Most of people that use `eval` want part of a variable name to be variable.

Once you realize that all global variables are held in the `window` object it becomes rather obvious that you can refer to that same field name without needing to use `eval` by referring to it.

```javascript
// Bad example - creates the variable name by concatenating constant and variable parts
eval('somevar' + num)
```

```javascript
// Good example - referencing the same field is simpler and more efficient
window['somevar' + num]
```

## Security risks

Beyond performance, `eval` is a significant **security vulnerability**. It can execute arbitrary code, making your application susceptible to **Cross-Site Scripting (XSS)** attacks if any part of the evaluated string can be influenced by user input or external data.

```javascript
// Dangerous - if userInput comes from an untrusted source, this is an XSS vector
eval('processOrder("' + userInput + '")')
```

For a deeper understanding of the risks, see the [OWASP XSS Prevention Cheat Sheet](https://cheatsheetseries.owasp.org/cheatsheets/Cross_Site_Scripting_Prevention_Cheat_Sheet.html).

## Content Security Policy (CSP)

**Content Security Policy** is a browser security mechanism that can restrict what scripts can run on your page. A strict CSP will include `'unsafe-eval'` in its blocked list, which prevents `eval`, `new Function()`, `setTimeout` with string arguments, and similar constructs from executing.

If your codebase relies on `eval`, it will break under a strict CSP — an additional reason to eliminate it.

## Safe alternatives

Instead of `eval`, prefer:

- **`window[key]`** — for dynamic property access on global objects
- **`JSON.parse()`** — for safely parsing JSON strings (never `eval` JSON)
- **`new Function()`** — slightly less dangerous than `eval` (runs in global scope, not local), but still avoid unless absolutely necessary
- **Lookup tables / Maps** — map string keys to functions or values explicitly

```javascript
// Instead of eval for dynamic dispatch, use a lookup table
const actions = {
  save: () => saveDocument(),
  print: () => printDocument(),
  export: () => exportDocument(),
};

actions[userAction]?.();
```
