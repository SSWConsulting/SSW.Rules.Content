The ideal development workflow is to clone the repository, make changes in Power BI Desktop, and follow the standard feature-branch and pull-request process.

However, when your admin need to make small Power BI report changes without waiting for a developer to become available, a Git-connected Development workspace provides a simple editing experience. They can edit and test reports in the shared workspace, commit changes without using Git commands or local branches, and send them for review before Production is updated.

<endIntro />

## What is a Power BI workspace?

A Power BI workspace is a shared location where users can view, edit, and manage reports, semantic models, and other Power BI items. Use a Development workspace for editing and testing. Keep the Production workspace for approved reports used by end users.

## How does a workspace work with Git

A workspace connects to one Git repository, one branch, and one repository folder at a time. Changes made in the workspace are compared with its connected branch:

## How to commit changes from workspace

1\. Open the Development workspace.

2\. Confirm nobody else is editing the same report.

3\. Edit, save, and test the report.

4\. Open \*\*Source control\*\*.

5\. Review the changed items.

6\. Enter a clear commit message.

7\. Select \*\*Commit\*\*.

8\. Send the change for review before Production is updated
