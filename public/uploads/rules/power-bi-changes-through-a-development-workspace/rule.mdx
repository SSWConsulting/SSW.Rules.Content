The ideal development workflow is to clone the repository, make changes in Power BI Desktop, and follow the standard feature-branch and pull-request process.

However, when an administrator needs to make a small Power BI report change without waiting for a developer, a Git-connected Development workspace provides a simple editing experience.

<endIntro />

## What is a Power BI workspace?

A Power BI workspace is a shared location where users can view, edit, and manage reports, semantic models, and other Power BI items. Use the workspaces as follows:

* **Development** – used by administrators to edit and test reports on a feature branch
* **Staging** – connected to main and used to validate approved changes
* **Production** – contains approved reports used by end users and is updated only through the deployment pipeline

## How does a workspace work with Git

A workspace connects to one Git repository, one branch, and one repository folder at a time. Changes made in the workspace are compared with its connected branch:

## How to commit changes from workspace

1\. Open the Development workspace.

2\. Create a feature branch from the latest main branch, switch the Sales Development workspace to the new feature branch.

3\. Edit, save, and test the report.

4\. Open Source control (see image above)

5\. Review the changed items.

6\. Enter a clear commit message, and commit the changes

7. Create a PR from the branch and wait for review

## How do you deploy Power BI changes using deployment pipeline?

<imageEmbed alt="power bi deployment stragegy" size="large" showBorder={false} figurePrefix="good" figure="Figure: Power BI deployment pipeline" src="/uploads/rules/power-bi-changes-through-a-development-workspace/powerbi-deployment-strategy.png" />

1. Merge your approved pull request to main
2. Open the Staging workspace and sync the changes from main branch
3. Test the updated reports in Staging workspace
4. Open the Power BI deployment pipeline.
5. Compare Staging with Production workspace and deploy the changes
6. Your changes is now live 🎉

Solution adapted from Microsoft Fabric’s official CI/CD workflow documentation. See [details](https://learn.microsoft.com/en-us/fabric/cicd/manage-deployment#option-3---deploy-using-fabric-deployment-pipelines)
