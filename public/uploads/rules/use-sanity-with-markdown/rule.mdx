Markdown works well in headless CMS platforms when content needs to be reusable across websites, apps, and multiple channels.

With [Sanity](https://www.sanity.io), the preferred approach is usually Portable Text for rich content, but Markdown can still be useful for technical or documentation-focused pages.

<endIntro />

## Recommended methods

* Use Portable Text for flexible rich content and editorial workflows
* Use Markdown fields only where portability is more important than rich editing
* Keep Markdown content structured and avoid unnecessary embedded HTML
* Define clear content models so editors know when to use Markdown vs rich content blocks

## Why this matters

This improves consistency and prevents content from becoming difficult to reuse across channels.

## Technical recommendation

If Markdown is the main source of truth for your content, [TinaCMS](https://tina.io) generally provides a cleaner workflow than adapting a block-based CMS for Markdown-heavy publishing.
