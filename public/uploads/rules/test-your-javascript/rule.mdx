Modern applications push significant business logic to the frontend. Whether you’re using React, Next.js, Angular, or another framework, your JavaScript or TypeScript code often handles validation, transformations, API calls, state management, and user flows.

If you don't use testing suites to reinforce the functionality of your code, you're relying on QA and hope.

Testing JavaScript/TypeScript applications is not optional. It's a part of building production-ready software.

<endIntro />

## Why testing matters (even with typescript)

TypeScript enforces type-safety, this improves safety but does not verify behaviour of logic.

Types check that your code compiles; it wont tell you:

* If a utility calculation function returns the wrong result 
* If the interaction between a button and user is incorrect
* If a state transition breaks a workflow

A combination of unit, integration and end-to-end tests will.

As AI driven development becomes more common in development workflows, it's essential that test suites are carefully crafted to ensure the developer's desired functionality. 

### Unit Tests

Unit testing involves verifying individual functions or components in complete isolate to ensure code reliability. These isolated sections of code should be the **smallest** **units** of code (for said functionality/ feature).

It is best practice to follow the [AAA Pattern](https://www.ssw.com.au/rules/how-to-structure-a-unit-test): Arrange, Act, Assert.

Unit tests can be written using: 

* 👑 [Vite](https://vite.dev/) (Recommended)
* [Jest](https://jestjs.io/)

Some examples of what Unit tests can be used for:

* Utility functions
* Business rules
* Data transformations

### Integration Tests

Stepping a level higher, integration involves testing that numerous components, functions and modules work together correctly to create a cohesive system. 

Testing behaviour rather than implementation - this helps find bugs between pieces of code, not inside them. Such examples include: 

* State updates are user interaction
* Form validation behaviour
* React components rendering with props

Integration Tests can be written using: 

* Vitest + Testing Library (i.e React Testing Library)
* Jest + Testing Library 

### End-to-End (E2E) Tests 

E2E tests aim to simulates real user interactions to validate the an entire flow within an application from start to finish. They're used to ensure that all components, from the UI, to backend and databases all work together as designed. 

Popular JavaScript E2E testing frameworks: 

* 👑 [Playwright](https://playwright.dev/) (recommended)
* [Cypress](https://www.cypress.io/#create)

Both of these frameworks allow you to simulate a a browser environment on the developer server, clicking buttons, inputting forms, and working through real-world workflows from the user's perspective.
