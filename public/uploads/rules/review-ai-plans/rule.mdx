Measure twice, cut once. Before any non-trivial change, always create **and review** a plan with AI. An AI plan is only as good as the human review it receives.

Reviewing the plan is what separates professional AI-assisted development from vibe coding. The Discuss step in the Plan → Discuss → Implement → Review workflow is where your domain knowledge and judgment provide the most value. See the rule on [AI-assisted development](https://www.ssw.com.au/rules/ai-assisted-development-workflow).

## Why plan review matters

Catching problems at the plan stage is dramatically cheaper than catching them after implementation (both in development costs as well as tokens):

* **Wrong assumptions** - the AI may have misunderstood requirements or made incorrect assumptions about your architecture
* **Scope creep** - the plan may include unnecessary changes that increase the scope of the change without you intending it to
* **Better alternatives** - you may have more domain specific knowledge and know of a more effective approach that the AI didn't consider
* **Multiple iterations** - reviewing and refining the plan leads to much better outcomes than approving the first draft

Modifying a plan is cheap, re-implementing a feature is expensive.

## The traditional approach

The original way to review AI plans is to discuss back and forth with your AI agent directly, such as in the terminal, or inside of your editor if you wanted to have more control by reading the raw `PLAN.md` file yourself.

### Terminal - Discussing and iterating with your AI Agent

The simplest approach to reviewing the plan is directly inside of the chat interface with your AI Agent, describing the modifications to your plan that you want.

### Editor - Overview of the actual plan file itself

For more direct control over just reading summaries inside of the terminal, you can open the generated plan file (e.g. `PLAN.md`) inside your editor alongside the chat session with your agent. This gives you a visual overview of the whole plan at once.

* **Edit manually** - revise the plan directly in the markdown file, then ask the AI to implement from the updated version
* **Cursor** - built-in plan mode shows the plan inline; you can chat about specific sections directly without leaving the editor
* **GitHub Copilot Workspace** - offers a plan-then-implement flow with a visual UI in the browser, where you can review and edit proposed changes before the AI implements them

### Limitations

These methods work, but they have clear drawbacks:

* Hard to give targeted feedback on specific sections without copying and pasting chunks of text
* No structured way to annotate, comment on, or reject parts of the plan
* Difficult to track what changed between plan iterations
* Suitable for small plans only - larger plans become unwieldy

## Tools that enhance plan review

A number of tools have popped up that give you a proper visual interface for reviewing and annotating AI plans, similar to how you would review a pull request on GitHub.

### Plannotator (Recommended)

[Plannotator](https://plannotator.ai/) provides a browser-based pull-request-style review UI - think GitHub PR review, but for AI plans. It makes structured, multi-round plan review practical for any plan size.

Key features:

* **Inline comments** on specific lines or sections (like GitHub PR review)
* **Delete** unwanted sections
* **Replace** or insert content
* **Global comments** for overall feedback
* **Multi-round iteration** - deny with annotations → AI refines → re-review → approve
* **Works with any AI agent** - Claude Code, Codex, Copilot CLI, etc.
* Once approved, the AI builds based on the agreed, refined plan

### UltraPlan

[UltraPlan](https://code.claude.com/docs/en/ultraplan) bridges your local Claude Code CLI with Claude Code on the web. You kick off a planning task from your terminal, and it drafts the plan in the cloud while your terminal stays free. Once ready, you review, comment on, and revise the plan in a rich browser-based UI.

Key features:

* **Inline comments** - highlight any passage in the plan and leave a comment
* **Emoji reactions** - react to sections to signal approval or concern
* **Outline sidebar** - navigate between plan sections easily
* **Terminal stays free** - the plan drafts remotely so you can keep working
* **Flexible execution** - approve to run on the web (auto-PR) or send the plan back to your terminal
* Launch via `/ultraplan` command, the keyword "ultraplan" in a prompt, or by promoting a local plan

### Claude Code VSCode Extension

The [Claude Code VSCode extension](https://marketplace.visualstudio.com/items?itemName=anthropic.claude-code) has built-in plan review features. When Claude generates a plan in VS Code, it automatically opens as a full markdown document in an editor tab - not just a chat message.

Key features:

* **Full document view** - the plan opens in a dedicated editor tab for easy reading
* **Inline comments** - add comments directly on specific sections of the plan to give Claude targeted feedback
* **Approval workflow** - once reviewed, choose to execute in auto mode, review each edit manually, or keep planning with feedback

<boxEmbed
  style="info"

    **Limitation:** Inline comments are freeform text feedback to Claude - there is no tracked resolution workflow like a GitHub PR review.
