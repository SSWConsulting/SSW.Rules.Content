Bundling and minification are essential techniques to improve JavaScript performance. They reduce the number of HTTP requests and the size of files delivered to the browser.

Modern JavaScript development has moved well beyond AMD and Gulp/Grunt — the ecosystem is now dominated by **ES Modules (ESM)** and next-generation bundlers.

<endIntro />

## ES Modules (ESM) — The Modern Standard

**ES Modules** (`import`/`export`) are natively supported in all modern browsers and Node.js. For new projects, always prefer ESM over CommonJS (`require`) or AMD:

```javascript
// ESM – the modern standard
import { formatDate } from './utils.js';
export function greet(name) { return `Hello, ${name}`; }
```

## Modern Bundlers

| Bundler | Best for | Key feature |
|---|---|---|
| **[Vite](https://vitejs.dev/)** | React, Vue, Svelte, vanilla | Lightning-fast dev server (native ESM), Rollup for production |
| **[Webpack](https://webpack.js.org/)** | Large enterprise apps | Highly configurable, huge ecosystem |
| **[Rollup](https://rollupjs.org/)** | Libraries and packages | Excellent tree-shaking, clean ESM output |
| **[Parcel](https://parceljs.org/)** | Zero-config projects | No config needed, good defaults |
| **[esbuild](https://esbuild.github.io/)** | Speed-critical builds | Extremely fast, often used inside other tools |

**Recommendation for new projects:** Start with **Vite**. It provides an instant dev server via native ESM and uses Rollup for optimised production builds.

## Tree-Shaking

Tree-shaking statically analyses your imports and removes unused code from the final bundle. This is only possible with ESM — it does not work with CommonJS.

```javascript
// Only `formatDate` is included in the bundle — `formatCurrency` is tree-shaken out
import { formatDate } from './utils.js';
```

Ensure your packages are ESM-first (check `"type": "module"` in `package.json` or `"exports"` field in library packages).

## Code Splitting

Modern bundlers support **code splitting** — dividing your app into smaller chunks that are loaded on demand. This reduces initial load time significantly for large applications.

```javascript
// Dynamic import – the chunk is only loaded when needed
const { HeavyComponent } = await import('./HeavyComponent.js');
```

Frameworks like React (with `React.lazy`), Next.js, and Angular handle code splitting automatically.

## Source Maps

Always enable **source maps** in development so browser DevTools display your original source code rather than minified output:

```javascript
// vite.config.js
export default {
  build: {
    sourcemap: true
  }
};
```

## Legacy: AMD and RequireJS

AMD (Asynchronous Module Definition) and [RequireJS](https://requirejs.org/) were popular before native ES Modules existed. **For new projects, do not use AMD.** Migrate existing AMD code to ESM.

**✅ Pro: Only the modules you need are downloaded**
**❌ Con: Each module requires a separate HTTP request (superseded by HTTP/2 multiplexing and bundlers)**

## Legacy: ASP.NET Bundling

ASP.NET's built-in bundling (`System.Web.Optimization`) and Gulp/Grunt task runners are legacy approaches. Modern .NET projects with React/Angular/Vue frontends should use Vite or Webpack for asset bundling, independent of the server stack.
