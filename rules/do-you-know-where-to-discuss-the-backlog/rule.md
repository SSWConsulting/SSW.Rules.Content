   </dt><dd>Figure: Bad Example – don't use emails to discuss tasks</dd></dl><h3>For code</h3><p>Sometimes developers need to discuss code implementations - sometimes to improve the code, other times to explain why something was done a certain way.<br></p><p>This should be done in the Pull Request, if possible comment directly on the line of the code change and once resolved, make sure that the important information is captured in the merge's commit description.</p><dl class="goodImage"><dt>
