<span class='intro'> ​​Code health is all about quality and consistency. Here is how to&#160;use various auditors and linters not just in your development environment, but also on your VisualStudio.com build.<br> </span>

<p>Depending on your development environment&#160;and the type of project, you should utilise different auditing tools.<br>&#160;1) Visual Studio - How to install and modify Visual Studio Analysers<br>&#160;2) Visual Studio Code - How to include TSLint and CSSLint extensions<br>&#160;3) VisualStudio.com - How to produce a build script which analyses code as part of the build process​​<br></p><p>Following the steps should take about 15 minutes to do, but longer to implement depending on the size of your solution.
