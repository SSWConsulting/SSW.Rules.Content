Considering the memory management of .NET Framework String.Empty will get higher performance then using "".

<!--endintro-->



```
public string myString 
   
{
 get
 {
 return ;
 } 
   
}
```




::: bad
Figure: Bad code. Low performance

:::



```
public string myString
{ 
   
 get 
   
 { 
   
 return string.Empty; 
   
 } 
   
}
```




::: good
Figure: Good code. Higher performance

:::
